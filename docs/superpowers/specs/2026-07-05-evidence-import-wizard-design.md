# Evidence Import Wizard Design

## Goal

Build one operator-facing command that validates filled host evidence manifests and beta response drafts before
records are imported. The default mode must not write records; writes are allowed only with an explicit import flag.

## Scope

The first version covers the current real adoption blocker:

- host manifests generated by `albu-mcp evidence session-manifest`
- beta drafts generated by `albu-mcp beta response-template`
- existing canonical record files: `docs/HOST_MANUAL_RUNS.json` and `docs/BETA_VALIDATION_RECORDS.json`
- a final gate hint based on `activation real-adoption-cycle`

The wizard does not create real evidence, fabricate beta outcomes, infer missing fields, or choose a product fix.

## Architecture

Create `src/albumentationsx_mcp/evidence_import_wizard.py` as a domain/orchestration module. It will load and validate
host manifests and beta drafts using existing primitives from `evidence.py`, `beta_validation.py`, and
`real_adoption_cycle.py`. The CLI adapter in `cli.py` will only parse arguments, call the module, format output, and
optionally write records when `--import-ready` is present.

## Command

Add:

```bash
albu-mcp evidence import-wizard \
  --host-manifest docs/operator-packets/codex-evidence-session-manifest.json \
  --host-manifest docs/operator-packets/claude-code-evidence-session-manifest.json \
  --beta-dir docs/beta-response-templates \
  --format json
```

Default behavior:

- validates inputs
- reports ready/blocked status per section
- reports exact blockers and next commands
- writes no records

Explicit write behavior:

```bash
albu-mcp evidence import-wizard \
  --host-manifest docs/operator-packets/codex-evidence-session-manifest.json \
  --host-manifest docs/operator-packets/claude-code-evidence-session-manifest.json \
  --beta-dir docs/beta-response-templates \
  --import-ready \
  --format json
```

`--import-ready` imports only when all supplied host manifests and beta drafts validate. If any input is invalid or
missing, the command exits with a validation error before writing.

## Output Contract

JSON output includes:

- `wizard_status`: `blocked`, `ready_to_import`, or `imported`
- `writes_records`: boolean
- `host_manifest_count`
- `beta_draft_count`
- `host_manifests`: per-manifest validation summaries
- `beta_drafts`: per-draft validation summaries
- `blocked_reasons`
- `next_commands`
- `post_import_cycle_status`

Text output is compact:

```text
evidence import-wizard blocked (host_manifests=0, beta_drafts=0, writes_records=false)
```

Markdown output is a short operator report.

## Error Handling

- Missing host manifest path is reported as a blocked host manifest item in no-write mode.
- Missing beta directory is reported as `beta_dir_missing` in no-write mode.
- Invalid JSON or schema validation errors are reported in the matching item.
- Import mode requires every supplied input to be ready; otherwise it raises `ValueError`.

## Testing

Use CLI tests with temporary records and temporary manifest/draft files:

- no-write mode does not change host or beta record files
- no-write mode reports missing inputs as blocked
- import mode writes valid host and beta records only when all inputs validate
- import mode rejects blocked inputs without partial writes

Run focused tests, ruff, ty, release readiness, and full CI before merge.
